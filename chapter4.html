III. DETAILED CONTENT

Section 4.1: Types of Software and Interrupts

4.1.1 System Software and Application Software

All computers begin as connected hardware items, and without software, these hardware items would be useless.

A. Software Hierarchy
There are two main types of software:
  • System Software: A set of programs to control and manage the operation of computer hardware. It provides a platform on which other software can run.
    Examples: Operating System (OS), Utility Programs, and Device Drivers.
  • Application Software (Apps): Software used to perform various applications on a computer. It allows a user to perform specific tasks.
    Examples: Word processor, Spreadsheet, Database, Video Editing software, and Control and Measuring software.
B. Examples of System Software (Utilities):
Utility programs are software designed to carry out specific tasks that help to manage, maintain, and control computer resources.
  • Anti-virus Software: Constantly checks for attacks, compares against a database, and may use heuristic checking. Infected files are put into quarantine.
  • Defragmentation Software: Rearranges blocks of data on an HDD to store files in contiguous sectors, leading to faster data access. (Not needed for SSDs).
  • Back-up Software: Allows scheduled back-ups. Recommended to have three versions: current, local back-up, and remote (cloud) back-up.
  • Security Software: Manages access control (user IDs, passwords), protects network interfaces (firewalls), and uses encryption/decryption.
  • Device Drivers: Software that enables hardware devices (e.g., a printer) to communicate with the operating system.

4.1.2 Operating Systems (OS)

The OS is essential software running in the background, managing basic functions and providing a useable interface between humans and the computer.

Key Functions of the OS:
  • Human Computer Interface (HCI): Allows interaction.
    • Command Line Interface (CLI): User types instructions.
      Advantage: User is in direct communication, not restricted.
      Disadvantage: User must learn many commands; slow and error-prone.
    • Graphical User Interface (GUI): Uses icons (WIMP/Post-WIMP).
      Advantage: More user-friendly, no need to learn commands.
      Disadvantage: Uses up considerably more computer memory than a CLI.
  • Memory Management: Manages RAM, keeps track of memory locations, and performs memory protection.
  • Security Management: Ensures integrity, confidentiality, and availability of data (updates, user privileges, access rights).
  • Hardware Peripheral Management: Communicates with I/O devices using device drivers; manages queues and buffers.
  • File Management: Performs tasks like creating/deleting files, maintains directory structures, and ensures access control.
  • Multitasking: Allows carrying out more than one task at a time by sharing hardware resources based on time limits and priority.
  • Management of User Accounts: Manages multiple users (user name, password), allowing customization and separate folders.

4.1.3 Running Applications (Firmware and BIOS)

  • Booting Up: When a computer starts, the bootstrap loader loads part of the OS into RAM.
  • BIOS (Basic Input/Output System): Handles the start-up of the motherboard.
    • The BIOS is often referred to as Firmware.
    • The BIOS program is stored in EEPROM (Electrically Erasable Programmable ROM).
    • BIOS settings are stored on a CMOS chip, powered by a battery.

4.1.4 Interrupts

An interrupt is a signal sent from a device or software to the microprocessor, causing it to temporarily stop what it is doing to service the interrupt.

Causes of Interrupts:
  • Input/Output processes (e.g., printer needs data).
  • Hardware faults (e.g., paper jam).
  • User interaction (e.g., pressing a key).
  • Software errors (e.g., divide by zero).
Operation of Interrupts:
  1. The computer identifies the interrupt type and priority.
  2. The status of the current task is saved (PC and registers).
  3. The Interrupt Service Routine (ISR) is executed.
  4. The status of the interrupted task is reinstated, and the process continues.
Buffers and Interrupts:
A buffer (temporary memory area) and interrupts are often used together. Data is sent to a buffer (e.g., for printing), and an interrupt signals the microprocessor to send more data when the buffer is empty, allowing the CPU to perform other tasks.

Section 4.2: Programming Languages, Translators, and IDEs

Programs must be translated into machine code (binary instructions) before a computer can use them.

4.2.1 High-Level and Low-Level Languages

A. High-Level Languages (HLL)
Characteristics: Independent of hardware; statements are closer to English. (e.g., Java, Python, C++).
Advantages: Easier to read/understand; quicker to write and debug.
B. Low-Level Languages (LLL)
Characteristics: Relate to specific hardware. (Machine Code, Assembly Language).
Advantages: Can use special hardware; uses less memory; performs tasks quickly.
Disadvantages: Takes longer to write/debug; harder to understand.
C. Assembly Language
A Low-Level Language that uses mnemonics (e.g., LDA, ADD, STO). It is translated using an Assembler.

4.2.2 Translators

A translator is a utility program that converts a program into machine code.

Translator Type Operation Key Advantages Key Disadvantages
Compiler Translates the entire HLL program (source code) into machine code (object code) all in one go. Compiled program runs faster. An executable file is produced that can be distributed and run without the compiler. Takes a longer time to write, test, and debug programs during development.
Interpreter Executes the HLL program one statement at a time, reading, translating, and performing the action. Easier and quicker to debug and test programs during development. Programs take longer to execute. Cannot be run without the interpreter.
Assembler Translates Assembly Language (LLL) into machine code. An executable file of machine code is produced. Only for use with Assembly Language.

4.2.3 Integrated Development Environment (IDE)

An IDE is a suite of programs used by programmers to aid the writing and development of programs.

Key Features of an IDE:
  • Code Editor: Allows programs to be written and edited.
  • Translator: Provides a compiler and/or interpreter.
  • Runtime Environment with Debugger: Allows single stepping, breakpoints, and shows variable contents.
  • Error Diagnostics and Auto-Correction: Dynamic error checking and suggested corrections.
  • Auto-Completion: Context-sensitive prompts for variables and reserved words.
  • Auto-Documenter: Explains the function and purpose of the code.
  • Prettyprinting: Code is colour-coded and formatted, making it easier to read.

Conceptual Summary: The Factory Metaphor

Imagine the computer system as a highly efficient factory.

  • The Operating System (OS) acts as the Central Factory Manager, managing all resources.
  • System Software is the underlying infrastructure (utilities, security, maintenance) that keeps the factory running.
  • Application Software is the specific task being performed (e.g., accounting, production planning).
  • An Interrupt is like an urgent alert buzzer (e.g., a machine jam or low raw materials supply). The Manager (CPU) must temporarily pause the current job, save its notes (registers), deal with the emergency (ISR), and then resume the original job.
  • A Buffer is a temporary storage bay on a conveyor belt. If the input (CPU) is fast and the output (Printer) is slow, the buffer holds the excess, preventing the Manager (CPU) from sitting idle.
