CHAPTER 8: PROGRAMMING

Subject: Computer Science (Core Programming Concepts, Data Structures, and File Handling)

Objectives Concepts Subroutines Data Structures File Handling Key Terms Activities

I. LEARNING OBJECTIVES

Upon completion of this chapter, the learner will be able to:

  • Understand and utilize fundamental programming concepts: sequence, selection (including nesting), iteration, totalling, counting, input, and output.
  • Define and use data storage: variables, constants, and appropriate data types (integer, real, char, string, Boolean).
  • Apply operators: arithmetic, logical, and Boolean operators.
  • Implement modular programming: define and use procedures and functions, including the use of parameters, local and global variables, and library routines.
  • Work with structured data: declare, use, read, and write arrays (one- and two-dimensional).
  • Perform file handling: store data in a file and retrieve data from a file.
  • Write maintainable programs.

II. DETAILED CONTENT (1/4): Programming Concepts

Writing computer programs puts your computational thinking to the ultimate test. Recommended languages include Python, Visual Basic, or Java.

8.1.1 Basic Constructs

  • Data use: Variables, constants, and arrays.
  • Sequence: The order of steps in a task.
  • Selection: Choosing a path through a program.
  • Iteration: Repetition of a sequence of steps.
  • Operator use: Arithmetic, logical, and Boolean.

8.1.2 Variables and Constants

Variable
A named data store containing a value that may change during program execution.
Constant
A named data store containing a value that does not change during program execution.
Declaration (Define)
Defines the value and data type. Some languages are explicit, others implicit. Python does not require separate declarations.

8.1.3 Basic Data Types

  • INTEGER: A positive or negative whole number.
  • REAL: A positive or negative number with a fractional part (decimal).
  • CHAR: A single character.
  • STRING: Several characters (letters, digits, symbols).
  • BOOLEAN: Can only have two values: TRUE or FALSE.

8.1.4 Input and Output

Input
Taken from a keyboard, must have a prompt. All inputs default as strings and may need casting (e.g., int(), float()).
Output
Results displayed on a screen, must have an explanatory message.

8.1.5 Sequence, Selection, and Iteration

Sequence
The order in which steps are executed is critical.
Selection
Allows choice of different paths (IF...THEN...ELSE or CASE statements).
Iteration (Looping)
Code repeated under certain conditions.
  • Count-controlled (FOR): For a set number of iterations.
  • Pre-condition (WHILE): Condition checked before; may have no iterations.
  • Post-condition (REPEAT...UNTIL): Always has at least one iteration; condition checked after.

8.1.6 Totalling and Counting

  • Totalling: TotalWeight = TotalWeight + Weight
  • Counting: NumberOfItems = NumberOfItems + 1 (or NumberOfItems++; in Java)

8.1.7 String Handling

  • Length: Finding the number of characters in the string.
  • Substring: Extracting a part of a string.
  • Upper: Converting all letters to uppercase.
  • Lower: Converting all letters to lowercase.

8.1.8 Operators

  • Arithmetic: +, -, *, /, ^ (Power).
    • MOD: Returns the remainder.
    • DIV: Returns the quotient (whole number part).
  • Logical: >, <, =, >=, <=, <> (Not equal).
  • Boolean: AND, OR, NOT.

8.1.9 Use of Nested Statements

Selection and iteration statements can be nested (included one inside the other) to reduce code and simplify testing.

II. DETAILED CONTENT (2/4): Procedures, Functions, and Libraries

Similar tasks are often grouped into subroutines, defined once and called many times.

8.2.1 Procedures and Functions

Procedure
A set of statements that performs a task but does not return a value. (void functions in Python, subroutines in VB, methods in Java).
Function
Similar to a procedure, but it always returns a value using the RETURN keyword.

8.2.2 Parameters

Parameters are variables in a procedure/function that store the values of the arguments passed from the main program. The call must match the definition.

8.2.3 Local and Global Variables

Global Variable
Can be used by any part of the program; its scope is the whole program.
Local Variable
Can only be used by the part it was declared in; its scope is restricted.

8.2.4 Library Routines

Fully tested functions and procedures provided by the language. Required routines include:

  • MOD: Returns the remainder.
  • DIV: Returns the quotient.
  • ROUND: Returns a value rounded to a given number of decimal places.
  • RANDOM: Returns a random number.

8.2.5 Creating a Maintainable Program

A maintainable program is one that can be easily updated or understood. It should have:

  • Meaningful identifier names.
  • Modular design (procedures and functions).
  • Full commenting (e.g., # in Python, ' in Visual Basic, // or /* */ in Java).

II. DETAILED CONTENT (3/4): Arrays

An array is a data structure containing several elements of the same data type, accessed using the same identifier name. The position is identified by an index (usually starting at zero).

8.3.1 One-dimensional Arrays

Often referred to as a list. Declaration (Pseudocode example):

DECLARE MyList : ARRAY[0:9] OF INTEGER

Arrays are typically populated using iteration (a FOR loop), using the loop counter as the array index.

8.3.2 Two-dimensional Arrays

Often referred to as a table (rows and columns). Declaration (Pseudocode example):

DECLARE MyTable : ARRAY[0:9,0:2] OF INTEGER

To populate a 2D array, two nested loops are required (one for rows, one for columns).

II. DETAILED CONTENT (4/4): File Handling

8.4.1 Purpose of Storing Data in a File

Data is stored in a file when it needs to be kept for future use. Data in RAM is volatile (lost when power is off), but data in a file is stored permanently.

8.4.2 Using Files

Every file is identified by its filename. Basic operations:

  1. Open: Open the file (specifying WRITE or READ).
  2. Read/Write: Use commands (e.g., WRITEFILE, READFILE) to transfer data.
  3. Close: Close the file (CLOSEFILE).

Analogy to solidify understanding

Mastering programming concepts is like learning to conduct an orchestra. The basic concepts (variables, constants, operators) are the fundamental instruments and notes. Sequence, selection, and iteration are the rhythm and structure of the composition. Procedures and functions are like pre-defined solos or musical themes that you can call upon repeatedly. Arrays and file handling serve as the storage (like a music library) and the sheet music stand, containing all the complex scores that you access to create a complete, logical, and maintainable performance.
