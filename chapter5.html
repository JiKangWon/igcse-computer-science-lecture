II. CORE CONTENT - PART 1: THE INTERNET AND THE WWW

1.1. Differences between the Internet and the WWW

The Internet (from INTERconnected NETwork) is a worldwide collection of interconnected networks and devices. It relies on physical infrastructure.

The World Wide Web (WWW) is only a part of the internet. It is a massive collection of multimedia web pages on websites, accessed using a web browser.

Internet World Wide Web (WWW)
Users can send and receive emails. It is a collection of multimedia web pages and other information on websites.
Allows online chatting (via text, audio, and video). http(s) protocols are written using Hypertext Mark-up Language (HTML).
Makes use of Transmission Protocols (TCP) and Internet Protocols (IP). URLs are used to specify the location of web pages.
It is a worldwide collection of interconnected networks and devices. Web resources are accessed by web browsers.

1.2. Uniform Resource Locators (URLs)

URLs are text addresses used to access websites. A URL is typed into a browser address bar using the format:

protocol://website address/path/file name

The website address includes the domain host (www), domain name, domain type (e.g., .com, .org), and sometimes the country code (e.g., .uk).

1.3. HTTP and HTTPS

HTTP (Hypertext Transfer Protocol)
A set of rules that must be obeyed when transferring files across the internet.
HTTPS (HTTP Secure)
Used when security (e.g., SSL or TLS) is applied. The ‘s’ stands for secure. It is often accompanied by the green padlock symbol.

1.4. Web Browsers

Browsers are software that allow a user to access and display web pages. Browsers interpret (translate) the HTML sent from websites and display the result.

Common browser features include:

  • Having a home page.
  • Storing favorite websites (bookmarks / favourites).
  • Keeping a user history of visited websites.
  • The ability to navigate forwards and backwards.
  • Allowing multiple tabs.
  • Making use of cookies and hyperlinks.
  • Storing data as a cache.

1.5. Retrieval and Location of Web Pages (DNS)

To retrieve a page, the browser needs the website's IP address. The Domain Name Server (DNS) is a system for finding IP addresses for a domain name. DNS eliminates the need for a user to memorize IP addresses.

The retrieval process:

  1. The user enters the URL; the browser asks a DNS server (1) for the IP address.
  2. If DNS (1) cannot find it, it asks another DNS server (2).
  3. DNS (2) finds the URL, maps it to the IP address (e.g., 107.162.140.19), and sends it back to DNS (1), which stores it.
  4. The IP address is sent back to the user’s computer.
  5. The computer communicates with the web server, and the HTML files are downloaded and interpreted by the browser.

1.6. Cookies

Cookies are small files or code stored on a user’s computer, sent by a web server. They contain (key, data) pairs (e.g., (surname, Jones)) to track users and maintain preferences.

Type of Cookie Session Cookie Persistent (Permanent) Cookie
Storage Stored in temporary memory. Stored on the hard drive until the expiry date or deleted.
Lifespan Cease to exist once the browser is closed. Remain in operation even after the browser is closed.
Usage Example Keeping items in a virtual shopping basket. Remembering a user’s log-in details.

Uses of Persistent Cookies (tracking cookies):

  • Allow the website to remember passwords, email addresses, etc.
  • Enable the website to recognize users every time they visit.
  • Track internet habits and users’ website histories/favorites.
  • Target users with advertising.
  • Store users’ preferences (e.g., customized web pages).

PART 2: DIGITAL CURRENCY

2.1. What is Digital Currency?

Digital currency exists purely in a digital format and has no physical form. It is an accepted form of payment. Traditional digital currency (e.g., PayPal, Apple Pay) relies on a central banking system, which poses confidentiality and security issues.

2.2. Cryptocurrency and Decentralization

Cryptocurrency is a form of digital currency that addresses security problems by introducing decentralization.

  • It uses cryptography (coding) to track transactions.
  • It has no state control; rules are set by the community.
  • Transactions are publicly available and monitored.
  • The system operates within a blockchain network, making it more secure.

2.3. Blockchaining

Blockchain is a decentralized database. All transactions are stored on this database. The network consists of interconnected computers, and all data is stored on all computers in the network. Data cannot be changed without the consent of all members, removing the risk of hacking.

How Blockchain Works:

  1. When a new transaction occurs, a new block is created.
  2. Each block contains:
    • Data (e.g., sender/recipient names, amount).
    • A unique Hash value (a fingerprint).
    • The Previous hash value (which points back to the preceding block).
  3. Block 1 is the genesis block (no previous hash).
  4. If data in a block is changed, its hash value changes. The subsequent block, relying on the old hash, becomes invalid, and the chain is broken, preventing tampering.
  5. The system uses proof-of-work to slow down the process (approx. 10 mins per block) to foil fast attacks.

PART 3: CYBER SECURITY

Keeping data safe (personal or commercial) is extremely important. Data can be corrupted or deleted through accidental damage or malicious acts.

3.1. Cyber Security Threats

Threat Description
Brute Force Attack Systematically trying all combinations of letters, numbers, and symbols to crack a password.
Data Interception Stealing data by tapping into a communication link (e.g., packet sniffing or wardriving).
DDoS Attack Attempting to prevent users from accessing a network by flooding it with useless spam traffic from many computers.
Hacking Gaining illegal, unauthorized access to a computer system.
Phishing Sending legitimate-looking emails to trick recipients into giving personal details or clicking fake links. (User must initiate action).
Pharming Malicious code that redirects a user’s browser to a fake website without their knowledge (e.g., DNS cache poisoning).
Social Engineering Manipulating people (exploiting emotions) into breaking security procedures (e.g., giving away a password).

3.2. Malware (Malicious Software)

Type of Malware Nature and Key Characteristics
Viruses Programs that replicate themselves. Need an active host program to run and cause damage (e.g., deleting files).
Worms Stand-alone malware that can self-replicate. Do not need a host program and can spread throughout a network without user action.
Trojan Horse Malicious program disguised as legitimate software. Needs to be executed by the end-user. Often installs other malware (Spyware, Ransomware).
Spyware Gathers information by monitoring user activities (e.g., web browsing, key logging) and sends data back.
Adware Attempts to flood the user with unwanted advertising (pop-ups).
Ransomware Programs that encrypt data and 'hold it hostage' until a ransom is paid. Best prevention is regular back-ups.

3.3. Solutions to Keep Data Safe

3.3.1. Access Levels and Authentication
Access Levels control a user’s rights (read, write, delete). Authentication is the ability of a user to prove who they are (e.g., password, biometrics, phone).
3.3.2. Passwords and Two-step Verification
Strong passwords should contain capitals, numbers, and special characters. Two-step verification (2FA) requires two methods of authentication (e.g., password + phone code).
3.3.3. Biometrics
Relies on unique human characteristics.
  • Fingerprint scans: Compares patterns of ridges. Benefits: unique, hard to replicate. Drawbacks: expensive, affected by injury.
  • Retina scans: Uses infrared to scan blood vessel patterns. Benefits: very high accuracy. Drawbacks: intrusive, slow, very expensive.
3.3.4. Firewalls and Proxy Servers
Firewalls (hardware or software) filter information in and out, blocking traffic that fails criteria. Primary defense against hacking.
Proxy Servers act as an intermediate between the user and a web server. They filter traffic, keep the user’s IP secret, and use a cache to speed up access.
3.3.5. Secure Sockets Layer (SSL) and Transport Layer Security (TLS)
SSL is a protocol that allows data to be sent securely (https, padlock symbol). It encrypts data. The web server sends an SSL certificate to authenticate the website.
TLS is a more modern and secure version of SSL.
3.3.6. Automatic Software Updates
Updates are vital as they often contain patches that improve software security (protecting against malware) or remove bugs.

III. SUMMARY & REVIEW

Analogy for Core Concept

To differentiate the Internet and the World Wide Web (WWW), you can imagine the Internet as the global postal and infrastructure system, consisting of all the physical wires, cables, routers, and the fundamental rules (TCP/IP) that allow information packets to travel anywhere. The World Wide Web is like the specific delivery service (like traditional mail or express shipping) that uses a certain format (HTML/HTTP) to send specialized documents (web pages) using that underlying infrastructure. The Internet is the road system; the WWW is the content delivery method that drives on those roads.
