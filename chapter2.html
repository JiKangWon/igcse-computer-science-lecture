II. DETAILED CONTENT

2.1 TYPES AND METHODS OF DATA TRANSMISSION

2.1.1 Data Packets

Data sent over long distances is usually broken up into data packets (datagrams), typically around 64 KiB. This makes them easier to control than a long continuous stream.

Benefits of Splitting Data

  • Each packet can be sent along a different route to its destination, useful if one route is busy or out of action.

Drawback

  • The necessity to reassemble the data when it reaches its destination.

Packet Structure

A typical packet is divided into three parts:

Part Content Purpose
Header IP address of the source and destination device, sequence number (for correct reassembly), and packet size. Essential for routing and ensuring all parts arrive intact.
Payload The actual data being sent in the packet (usually about 64 KiB). Carries the data component of the message.
Trailer A method of identifying the end of the packet and an error checking method. Enables separation of packets and verification of data integrity.

Error Checking in the Trailer (CRC): Cyclic Redundancy Checks (CRCs) are common. The sending computer adds up all the 1-bits in the payload and stores this total (as a hex value) in the trailer. The receiving computer recalculates this sum; if the two values match, no transmission errors have occurred.

2.1.2 Packet Switching

Packet switching is a method where a message is broken into multiple packets, and each packet is sent independently from the start point to the end point.

  • Role of the Router: At each node, a router receives the packet and determines the shortest possible route for it to take next based on the destination IP address.
  • Packet Route and Arrival: Packets often follow different paths and may arrive out of order. The receiving computer uses the sequence number in the header to reassemble them.
  • Hopping/Hop Number: A hop number can be added to the header, reduced by 1 at each router. If it reaches zero, the packet is deleted to prevent network clogging.

Benefits of Packet Switching

  • No need to tie up a single communication line.
  • Possible to overcome failed, busy, or faulty lines by re-routing.
  • Relatively easy to expand package usage.
  • High data transmission rate is possible.

Drawbacks of Packet Switching

  • Packets can be lost and need to be re-sent.
  • More prone to errors with real-time streaming (e.g., live events).
  • A delay occurs at the destination while packets are re-ordered.

2.1.3 Methods of Data Transmission (Serial, Parallel, Simplex, Duplex)

A. Transmission Modes (Direction):

Simplex
Data can be sent in ONE DIRECTION ONLY (e.g., computer to a printer).
Half-duplex
Data can be sent in BOTH DIRECTIONS but NOT AT THE SAME TIME (e.g., a walkie-talkie).
Full-duplex (Duplex)
Data can be sent in BOTH DIRECTIONS AT THE SAME TIME (simultaneously) (e.g., a broadband internet connection).

B. Data Transmission Types (Bit Flow):

Feature Serial Data Transmission Parallel Data Transmission
Mechanism Sends ONE BIT AT A TIME over a SINGLE WIRE/CHANNEL. Sends SEVERAL BITS (usually one byte) down SEVERAL CHANNELS/WIRES simultaneously.
Speed/Distance Slower rate than parallel. Works well over long distances. Faster method. Works well over short distances (e.g., internal computer circuits).
Synchronization Data arrives fully synchronised. Data can become skewed (out of synchronisation) over longer distances (>20m).
Example Connecting a computer to a printer via a USB connection. Internal circuits in a computer.

2.1.4 Universal Serial Bus (USB)

USB is a form of serial data transmission that has become the industry standard. It supports both half-duplex and full-duplex transmission.

Benefits of the USB System

  • Devices plugged into the computer are automatically detected.
  • The appropriate device driver software is automatically recognised and loaded.
  • It has become an industry standard.
  • Supports different data transmission rates (from 1.5 Mbps to 5 Gbps).
  • No need for an external power source, as the cable supplies +5 V power.
  • The USB protocol notifies the transmitter to re-transmit data if errors are detected.
  • It is backward compatible.

Drawbacks

  • Standard USB supports a maximum cable length of 5 meters.

2.2 METHODS OF ERROR DETECTION

2.2.1 The Need to Check for Errors

When data is transmitted, it risks being corrupted, lost, or gained due to interference, packet switching problems, or data skewing. Checking is vital as computers cannot understand corrupted data.

2.2.2 Parity Checks, Checksum, and Echo Checks

Parity Checks
Based on the number of 1-bits in a byte (EVEN or ODD). A parity bit is set to ensure the byte conforms to the protocol.
Limitation: Cannot detect if two bits change, and doesn't know *which* bit is in error.
Parity Blocks: Overcomes the limitation by checking both horizontally and vertically, allowing the position of the error to be identified.
Checksum
The sending computer calculates a checksum value from a block of data. This is sent with the data. The receiving computer recalculates the checksum. If they don't match, a re-send is requested.
Echo Check
Data sent to another device is sent back immediately to the sender for comparison.
Unreliability: If differences exist, it's unknown if the error was in the original or the return transmission.

2.2.3 Check Digits

A check digit is the final digit in a code, calculated from all other digits.

  • Purpose: To detect errors in data entry (not data transmission).
  • Errors Detected: Incorrect digits, transposition errors, or omitted/extra digits.
  • Examples: ISBN 13 and Modulo-11.

2.2.4 Automatic Repeat Requests (ARQs)

ARQ is a method using acknowledgements and timeout to check for errors.

  1. The receiving device uses an error code (like CRC).
  2. If no error, a positive acknowledgement is sent.
  3. If an error is found, a negative acknowledgement is sent, requesting re-transmission.
  4. The sender uses a timeout. If no acknowledgement is received within this time, it automatically re-sends the data.

2.3 ENCRYPTION

2.3.1 The Purpose of Encryption

When data is transmitted over a public network, it risks interception by hackers (eavesdroppers). Encryption alters data into an unreadable form to minimize this risk.

Plaintext
The original data before encryption.
Ciphertext
Encrypted data after passing through an encryption algorithm.

2.3.2 Symmetric and Asymmetric Encryption

A. Symmetric Encryption
Uses ONE SINGLE KEY to both encrypt and decrypt the message.
Major Drawback: The key must be sent to the recipient, meaning it can be intercepted.
B. Asymmetric Encryption
Uses two mathematically linked keys: the Public Key and the Private Key.
Public Key: Made available to everybody.
Private Key: Known only to the computer user.
Mechanism: To send data to Jane, Tom uses Jane’s Public Key to encrypt it. Jane then uses her matching Private Key to decrypt it.
Security: More secure because the Private Key is never transmitted.

III. SUMMARY AND REVIEW QUESTIONS

Key Concepts Summary

  • Data is broken into packets (Header, Payload, Trailer) and sent using packet switching.
  • Transmission modes include Simplex, Half-duplex, and Full-duplex (direction).
  • Transmission methods are Serial (one bit at a time, good for long distance, e.g., USB) and Parallel (multiple bits simultaneously, fast over short distances, prone to skewing).
  • Error checking methods following transmission include Parity Checks, Checksum, and Echo Checks.
  • ARQ uses acknowledgements and timeout to ensure data integrity.
  • Check Digits are used to detect errors during data entry.
  • Encryption (Plaintext to Ciphertext) protects data from eavesdroppers.
  • Symmetric encryption uses one key; Asymmetric encryption uses a Public Key (for encrypting) and a Private Key (for decrypting).

Review Questions

  1. Describe the three main components of a data packet and the function of each component.
  2. Explain the key difference between a check digit and a checksum.
  3. Describe how the Automatic Repeat Request (ARQ) process works, making reference to acknowledgement and timeout.
  4. Explain why asymmetric encryption is considered more secure than symmetric encryption.
  5. State whether an internal computer circuit (IC) that sends multiple bits simultaneously uses serial or parallel data transmission. Justify your answer based on distance and speed requirements.

Analogy for Asymmetric Encryption

Asymmetric encryption is like having a public mailbox and a private key to unlock it. Anyone can use your Public Key (which is your mailbox slot, known to everyone) to put a message (encrypted data) inside. However, only you hold the Private Key (the physical key) that can open the box and read the contents. The private key is never shared or transmitted, ensuring security.
