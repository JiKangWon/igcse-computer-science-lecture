3.1 Computer Architecture

Central Processing Unit (CPU)

The Central Processing Unit (CPU), also called a microprocessor or processor, is essential to all modern computer systems, including smartphones and tablets. The CPU is responsible for executing or processing all instructions and data within a computer application. The CPU generally consists of the Control Unit (CU), the Arithmetic and Logic Unit (ALU), registers, and buses.

Von Neumann Architecture

Developed in the mid-1940s by John von Neumann, this architecture introduced the concept of the 'stored program computer'. Its main features include:

  • The CPU concept.
  • The ability of the CPU to access memory directly.
  • The storage of programs and data in computer memory.
  • The sequential execution of stored program instructions.

Key CPU Components and Functions:

Arithmetic & Logic Unit (ALU)
Carries out required arithmetic (e.g., +, -, shifting) or logic operations (e.g., AND, OR).
Control Unit (CU)
Reads instructions from memory (whose address is stored in the Program Counter, PC), interprets them using the Fetch–Decode–Execute cycle, and generates signals along the control bus to synchronize data flow and program instructions throughout the computer.
Registers
These are fundamental, temporary components in the von Neumann system. Specific purpose registers include:
  • Program Counter (PC): Stores the address where the next instruction to be read can be found.
  • Memory Address Register (MAR): Stores the address of the memory location currently being read from or written to.
  • Memory Data Register (MDR): Stores data that has just been read from memory or data about to be written to memory.
  • Current Instruction Register (CIR): Stores the current instruction being decoded and executed.
  • Accumulator (ACC): Used during ALU calculations to store data temporarily.

System Buses

System buses act as parallel transmission components connecting major computer components.

Address Bus
Carries addresses throughout the system. It is unidirectional (travels in one direction only) between the CPU and memory. The width of the address bus determines how many memory locations can be directly addressed (e.g., 32 bits allows 4,294,967,296 locations).
Data Bus
Carries data (which can be an address, instruction, or numerical value). It is bidirectional, allowing data transfer between the CPU and memory/I/O devices.
Control Bus
Carries control signals from the Control Unit to all other computer components. It is typically 8-bits wide and is bidirectional.

Fetch–Decode–Execute Cycle

This is the process used by the CPU to carry out a set of instructions:

  1. Fetch: The next instruction is fetched from the memory address stored in the MAR and copied into the MDR, and then into the CIR. The PC is then incremented by 1.
  2. Decode: The instruction is interpreted.
  3. Execute: The CPU passes the decoded instruction as control signals (via the control bus) to appropriate components to carry out the instruction.

CPU Performance Factors

Performance is influenced by:

System Clock
Produces timing signals for synchronization. Increasing clock speed (measured in GHz) potentially increases processing speed. Overclocking can lead to synchronization problems and overheating.
Bus Width
Increasing the width of the address and data buses increases system performance and speed.
Cache Memory
Located within the CPU, cache (using fast Static RAM/SRAM) stores frequently used data/instructions, resulting in much faster access times than RAM, thus improving performance.
Cores
A core is a unit made up of an ALU, a control unit, and registers. Using multiple cores (e.g., dual core, quad core) alleviates the continuous need to increase clock speeds.

Embedded Systems

An embedded system is a combination of hardware and software designed to execute a specific set of functions, often feedback-oriented. Examples include engine management, ABS braking, and GPS in motor vehicles.

Benefits

  • Small size, relatively low cost.
  • Low power consumption.
  • Dedicated to one task (simple interfaces).
  • Fast reaction to changing input (real time).

Drawbacks

  • Difficult to upgrade some devices.
  • Troubleshooting requires specialists.
  • Risk of hacking if connected to the internet.
  • Potentially contributing to a 'throw away' society.

3.2 Input and Output Devices

Input Devices

These devices convert external data into a form the computer can understand and process.

Barcode and QR Code Scanners
Barcodes use parallel lines. QR codes use a matrix of squares, hold more information (up to 7089 digits), and can be read by smartphone cameras.
Digital Cameras
Capture images when light hits a light-sensitive cell (CCD) made of pixels. An Analogue to Digital Converter (ADC) forms the digital image.
Keyboards
Each keypress completes a circuit, and the CPU interprets the character based on its ASCII value.
Microphones
Convert sound waves into an analogue electric current, which is then digitized by a sound card (using an ADC).
2D and 3D Scanners
2D scanners input hard copy documents (may use OCR). 3D scanners scan solid objects, capturing x, y, z coordinates, often for CAD.
Touch Screens
Allow simple touch selection. Three types exist:
  • Capacitive: Uses the change in the screen's electrostatic field when touched by a conductor (finger).
  • Infrared: Uses an array of infrared beams; touching breaks the beams.
  • Resistive: Made of two layers; pressure brings them into contact, completing a circuit.

Output Devices

These devices show the results of computer processing in a human-understandable form.

Actuators
Mechanical or electromechanical devices (like solenoids or motors) used to control devices, converting an electrical signal into mechanical movement.
Light Projectors (DLP/LCD)
  • DLP (Digital Light Projector): Uses millions of micro mirrors on a DMD chip.
  • LCD (Liquid Crystal Display) Projector: Passes a high-intensity light beam through LCD screens (red, green, blue).
Printers (Inkjet/Laser/3D)
  • Inkjet Printers: Spray droplets of liquid ink.
  • Laser Printers: Use dry powder ink (toner) and static electricity. Fast for high volume.
  • 3D Printers: Produce solid objects layer by layer (additive manufacturing).
Screens (LED/LCD/OLED)
  • LCD Screens: Use tiny liquid crystals and require backlighting (often LED).
  • LED Screens: Made of tiny Light Emitting Diodes (red, green, blue).
  • OLED (Organic LED): Uses flexible organic materials, generates its own light (no backlighting), making them thinner and lighter.
Loudspeakers
Convert a digital signal (which is first passed through a Digital to Analogue Converter, DAC) into sound waves by causing a cone to vibrate.

Sensors in Control and Monitoring

Sensors are input devices that measure physical properties (analogue data). This data must be converted using an ADC.

Monitoring System
The microprocessor analyzes data and sends a warning if it's outside the acceptable range, but does not affect the process (e.g., monitoring patient vital signs).
Control System
If data is outside the range, the computer sends signals (often via a DAC to actuators) which affect the next set of inputs, creating a feedback loop (e.g., controlling central heating).

3.3 Data Storage

Primary Memory

Memory accessible directly by the CPU.

RAM (Random Access Memory)
Volatile (contents lost when power is off). Temporary memory for data and programs currently in use.
  • DRAM (Dynamic RAM): Consists of transistors and capacitors; must be constantly refreshed.
  • SRAM (Static RAM): Uses flip flops, faster than DRAM, and used in CPU cache memory.
ROM (Read-Only Memory)
Non-volatile (contents are permanent). Data cannot be changed, often stores the BIOS or start-up instructions.

Secondary Storage

Non-volatile storage devices not directly addressable by the CPU, used for permanent storage.

Magnetic Storage (HDD)
Hard Disk Drives store data on magnetic platters, organized into tracks and sectors. Suffers from latency and fragmentation.
Solid State Storage (SSD)
No moving parts, resulting in much faster data access. Use NAND/NOR chips. Consume less power and run cooler. Memory sticks also use this technology.
Optical Storage (CD/DVD/Blu-ray)
Uses laser light to read and write data.
  • DVDs have larger capacity than CDs (smaller pits).
  • Blu-ray Discs use a blue laser (shorter wavelength) for even smaller pits, enabling greater storage (e.g., 50 GB).

Virtual Memory

A memory management system using secondary storage (HDD/SSD) to handle situations when physical RAM runs out. A key drawback is disk thrashing—a high rate of data swapping that can severely reduce processing speed.

Cloud Storage

Data is stored on remote servers managed by a hosting company. Types include Public, Private, and Hybrid cloud.

Benefits

  • Access from anywhere.
  • No need for external storage.
  • Remote back-up.

Drawbacks

  • Reliance on a stable internet connection.
  • High costs for large capacity.
  • Relinquishing control over data security.

3.4 Network Hardware

Network Interface Card (NIC)
A necessary hardware component (circuit board or chip) that allows a device to connect to a network. Wireless NICs (WNICs) use wireless connectivity.
MAC Address (Media Access Control)
A unique 48-bit identifier (e.g., NN–NN–NN–DD–DD–DD). It identifies the physical address of a device on the network.
  • Universally Administered (UAA): Set by the manufacturer.
  • Locally Administered (LAA): Can be changed by the user.
IP Address (Internet Protocol)
Protocols define the rules for data communication. An IP address identifies the global address of a device connected to the internet.
  • IPv4: Based on 32 bits.
  • IPv6: Uses 128-bit addresses for a larger number of hosts.
  • Dynamic IP Address: Temporary, assigned by a DHCP server.
  • Static IP Address: Permanently assigned, used for servers.
Routers
Devices that route data packets between different networks (e.g., LAN to WAN). They inspect the IP address of a data packet to send it to the appropriate switch, which then uses the MAC destination address to deliver it.

Analogy for CPU Components and Data Flow

Imagine the CPU is a small, high-speed kitchen working on a recipe (the program).

  • The ALU is the chef performing all the cuts, mixing, and cooking (arithmetic and logic operations). The ACC is the temporary bowl used right next to the chef.
  • The Control Unit (CU) is the supervisor. It looks at the recipe steps, coordinates the workflow, and sends instructions to everyone else.
  • The Program Counter (PC) is the bookmark marking which step of the recipe is next. The MAR is the address label of the cupboard/fridge containing the ingredient. The MDR is the item retrieved from that location. The CIR is the copy of the current step the supervisor is reading out loud.
  • The Fetch–Decode–Execute cycle is the continuous loop of reading the next step, understanding it, and performing the action.
  • The System Clock is the steady drum beat dictating the speed and rhythm.
  • The Buses are the conveyor belts: the Address Bus specifies where to go (unidirectional), the Data Bus carries the item (bidirectional), and the Control Bus carries the supervisor's signals.
